# STRATEGY_SPEC.md

このリポジトリの**北極星**。「何を解くか・どう進めるか・何で成否を測るか」を1枚に固定する。
作業サイクル（plan → work → review → learning）の開始時にここへ立ち返り、個別判断がこの目的・方針・指標から外れていないかを確認する。

頻繁には変えない。変える場合は理由を添えて更新する。作業手順は `CLAUDE.md`、得られた学びは `.learnings/` を参照。

## プロジェクトの目的

macOS 開発環境を**再現可能・一貫した形で**管理する dotfiles。シェル（zsh）・エディタ（Vim/Neovim）・各言語ツールチェイン・システム環境設定を、自動インストールとシンボリックリンクで集約し、新規端末でも同じ環境を素早く立ち上げられる状態を保つ。

AI コーディングエージェント（Claude Code / Codex / Pi 等）の設定資産も、複数ツールで共有できる形でこのリポジトリに集約する。

## 方針

- **Single Source of Truth**: 設定はリポジトリに集約し、`symlink.sh` でホームへリンクしてリポジトリを唯一の正とする。
- **Declarative over Imperative**: 「あるべき状態」を宣言（`*.yml` 等）し、実行はスクリプト/ランナーに委ねる。
- **自動化の最大化 + Human in the Loop**: 可能な限り自動化しつつ、セキュリティ・個人情報など要所では人の確認を介在させる。
- **速度と効率の最適化**: 操作の応答速度・キーストロークを削る方向で設計する。
- **モジュール性と拡張性**: 機能は疎結合に分割（`scripts/` の言語別スクリプト等）。
- **Convention over Configuration**: 標準ツール・規約（mise 等）を採用し設定の複雑さを抑える。
- **Cross-Agent Portability**: AI エージェント設定はポータビリティで階層化し、可搬な層（context file / Agent Skill / plain Markdown）へロジックを寄せ、ツール固有の接着剤は最小化する。
- **テストの分離**: テストはシンボリックリンク先を汚さないよう repo 直下 `tests/{language}/` に隔離する。

## 主要指標

- **再現性**: クリーンな macOS で `./install.sh`（= `task install:all`）が最後まで完走し、新規端末で同等の環境が立ち上がること。
- **テスト**: `tests/` 配下のテスト（`Taskfile.tests.yml`）が通ること。
