# セッション振り返り

このセッションで行った作業を振り返り、今後のために記録すべき知見を整理します。

$ARGUMENTS

## 振り返りプロセス

### 1. セッションサマリー作成

まず、このセッションで実施した作業の概要を作成します：

- 実施した主な作業内容
- 変更したファイルや機能
- 直面した課題や解決した問題
- 使用したツールや技術

### 2. 抽象化フェーズ

具体的な作業内容から、汎用的な学びや知見を抽出します：

**重要な原則：**
- 局所最適化にならないように注意する
- システム全体との整合性やバランスを意識する
- ただし、具体的な細かい内容でも重要な内容は例外的にOKとする

抽出する知見の例：
- 設計上の判断基準や理由
- 技術選定の背景
- 陥りやすい落とし穴とその回避方法
- パフォーマンスやセキュリティに関する考慮事項
- 開発効率を向上させる手法

### 3. ステアリングドキュメント反映候補の抽出

プロジェクトの方針やルールとして記録すべき内容を特定します：

- 新しく発見したベストプラクティス
- プロジェクト固有の制約や要件
- チーム内で共有すべき知見
- 将来の開発で参照すべきガイドライン

### 4. 承認と反映

**AskUserQuestion ツールを使用して、メニュー形式で候補を提示します：**

各候補について：
- 候補の内容を簡潔に説明
- どのステアリングドキュメントに反映すべきか提案
- 反映する価値がある理由を明記

ユーザーが選択した候補のみを `.kiro/steering/` 配下の適切なファイルに反映します。

## 出力フォーマット

振り返りレポートは `.kiro/steering/retrospectives/` ディレクトリに保存します：

```
# セッション振り返り - [YYYY-MM-DD]

## 📋 セッションサマリー

[このセッションで実施した作業の概要]

## 💡 抽象化された学び・知見

### [学び1のタイトル]
[詳細な説明]

### [学び2のタイトル]
[詳細な説明]

## 🎯 ステアリング反映候補

### 候補1: [タイトル]
- **対象ファイル**: `.kiro/steering/[ファイル名].md`
- **内容**: [反映すべき内容]
- **理由**: [なぜこれを記録すべきか]

### 候補2: [タイトル]
...
```

## 重要事項

- このセッションで新たに得られた知見のみを抽出してください
- 既にステアリングドキュメントに記載されている内容は除外してください
- 将来のセッションで役立つ情報を優先的に抽出してください
- Less is More の原則に則り、簡潔で本質的な内容に絞る
