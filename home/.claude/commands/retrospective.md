# セッション振り返り

このセッションで行った作業を振り返り、今後のために記録すべき知見を整理します。

$ARGUMENTS

## 振り返りプロセス

### 1. セッションサマリー作成

まず、このセッションで実施した作業の概要を作成します：

- 実施した主な作業内容
- 変更したファイルや機能
- 直面した課題や解決した問題
- 使用したツールや技術

### 2. 抽象化フェーズ

具体的な作業内容から、汎用的な学びや知見を抽出します：

**重要な原則：**
- 局所最適化にならないように注意する
- システム全体との整合性やバランスを意識する
- ただし、具体的な細かい内容でも重要な内容は例外的にOKとする

抽出する知見の例：
- 設計上の判断基準や理由
- 技術選定の背景
- 陥りやすい落とし穴とその回避方法
- パフォーマンスやセキュリティに関する考慮事項
- 開発効率を向上させる手法

### 3. 永続化候補の抽出

プロジェクトの方針やルールとして記録すべき内容を特定します：

- 新しく発見したベストプラクティス
- プロジェクト固有の制約や要件
- チーム内で共有すべき知見
- 将来の開発で参照すべきガイドライン

知見の種別に応じて保存先を判断します：

| 種別 | 保存先 |
|------|--------|
| プロジェクト全体のルール・方針（バージョン管理・共有が必要なもの） | `CLAUDE.md` |
| トピック別ルール・特定ファイルパスにスコープするルール | `~/.claude/rules/*.md`（個人）または `.claude/rules/*.md`（プロジェクト共有） |
| ユーザープロファイル（役割・スキル・好み） | auto-memory `user` 型 |
| アプローチへのフィードバック（修正・承認されたやり方） | auto-memory `feedback` 型 |
| プロジェクト状況・意思決定・背景 | auto-memory `project` 型 |
| 外部リソース参照（ドキュメント・ツールの場所） | auto-memory `reference` 型 |

### 4. 承認と反映

**AskUserQuestion ツールを使用して、メニュー形式で候補を提示します：**

各候補について：
- 候補の内容を簡潔に説明
- 保存先（`CLAUDE.md` / `.claude/rules/` / `~/.claude/rules/` / auto-memory の型）を提案
- 反映する価値がある理由を明記

ユーザーが選択した候補のみを反映します。

## 出力フォーマット

振り返りレポートは、このセッションのシステムコンテキストで Claude Code が提示している
auto-memory ディレクトリ（"You have a persistent, file-based memory system at ..." として示されるパス）
配下の `retrospectives/` サブディレクトリに保存します。

**注意**: カレントプロジェクトのディレクトリ内ではなく、Claude Code が管理する
`~/.claude/projects/[current-project-hash]/memory/retrospectives/` です。

保存後は同ディレクトリの `MEMORY.md` にエントリを追記してください：
```
- [セッション振り返り YYYY-MM-DD](retrospectives/YYYY-MM-DD.md) — [一行サマリー]
```

振り返りレポートのファイル形式（auto-memory 準拠）：

```markdown
---
name: セッション振り返り YYYY-MM-DD
description: [一行サマリー — MEMORY.md での関連性判断に使われるため具体的に]
type: project
---
```

## 重要事項

- このセッションで新たに得られた知見のみを抽出してください
- 既に CLAUDE.md や auto-memory に記載されている内容は除外してください
- 将来のセッションで役立つ情報を優先的に抽出してください
- Less is More の原則に則り、簡潔で本質的な内容に絞る
