# Takeover - セッション情報の復元

`.claude/HANDOVER.md` から前回のセッション情報を読み込み、状況を把握して次のアクションを提案します。

## 実行手順

### 1. ハンドオーバーファイルの確認

- `.claude/HANDOVER.md` が存在するか確認
- ファイルが存在しない場合:
  - ユーザーに「HANDOVER.md が見つかりません」と報告
  - 新規セッションとして開始するか、`/handover` で作成するか提案
  - 処理を終了

### 2. ファイル読み込み

- Read ツールを使用して `.claude/HANDOVER.md` の全内容を読み込む

### 3. 内容の分析と理解

ハンドオーバードキュメントを詳細に分析し、以下を把握する:

- 前回セッションの作業内容と文脈
- 達成された成果と未完了のタスク
- 重要な技術的決定と背景
- 遭遇した問題と解決策
- 学んだ教訓と注意点
- 重要なファイルとその役割

### 4. 状況報告

ユーザーに以下の形式で状況を報告:

#### 📋 前回セッションのサマリー

- 取り組んでいたタスク
- 達成されたこと
- 主要な成果物

#### ✅ 完了した項目

- 実装された機能
- 解決されたバグ
- 完了したタスク

#### ⚠️ 注意が必要な事項

- 未解決の問題
- 既知の制限事項
- 注意すべき gotchas

#### 💡 前回の重要な決定

- 技術選択とその理由
- アーキテクチャ判断
- トレードオフ

#### 🗺️ 重要なファイル

- 前回変更されたファイル
- 今後変更が必要なファイル
- キーとなるコードの場所

#### 🎯 提案された次のステップ

- 前回提案された次のアクション
- 優先順位順にリスト化
- 各アクションの依存関係

### 5. 次のアクション提案

前回の HANDOVER.md の内容に基づき、具体的な次のアクションを提案:

- 続きから作業を再開する選択肢
- 前回の問題を解決する選択肢
- 新しい方向性を探る選択肢
- ユーザーに「どこから始めますか？」と尋ねる

アルファベット番号付き箇条書きで提示（a, b, c, d など）

### 6. コンテキストの提供

- 必要に応じて、重要なファイルを Read して最新の状態を確認
- 前回のセッション以降に変更があった可能性を考慮
- ユーザーに追加の文脈が必要か確認

## 重要な注意事項

- HANDOVER.md の内容を正確に理解する
- 前回の Claude の意図や文脈を尊重する
- ユーザーに状況を明確に伝え、次のアクションを選びやすくする
- 前回からの継続性を保ちつつ、新しい視点も提供する
- 必要に応じて、前回の決定や方針について再確認する
