### スキル作成/改善時は散文化された決定論処理をスクリプトへ寄せられないか点検する

**学び**: skill の手順を書く・見直すとき、毎回モデルが手作業で行う **決定論的な処理**（存在チェック、ディレクトリ作成、テンプレ/ファイルのコピー、スキャフォルド生成）が散文で書かれていたら、それはスクリプト化の候補。スクリプトへ寄せると (1) 手順が「スクリプトを1回実行する」という不可避・冪等なアクションになり、ゲートの素通りや手作業のブレが消え、(2) モデルには判断と要確認だけが残る。分担の境界は明確で、**「CLAUDE.md 等の“要ユーザ確認”編集」と「docs から読み取る等の LLM 判断」はモデル / それ以外の機械的処理はスクリプト**。スクリプトは `SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"` で自己位置からテンプレを相対参照すれば agent 固有変数（`${CLAUDE_SKILL_DIR}` 等）に依存せず、skill ディレクトリごとシンボリックリンクされていても動く。出力は「今回作成した分だけ ACTION NEEDED」を出し、すべて既存なら `all set` にすると、毎回の再提案を防げる。

**根拠**: 2026-06-26、learning skill の section 0 をスクリプト化した作業。存在チェック＋テンプレコピーが散文だったのを `scripts/setup.sh` に集約し、使い捨て HOME・一時 git repo・シンボリックリンク経由で「冪等性・部分補完・非git skip・自己位置解決」を確認した。

**適用範囲**: 冪等な setup/bootstrap を持つ、または決定論的な反復処理を含む agent skill の新規作成・改善（skill-creator / skill-refiner を使うときの点検観点）。ゲートの書き方そのものは同じ `.learnings/skill-setup-existence-gate.md` を併せて参照。スキル作成は他プロジェクトでも起きるが、本リポジトリが skill 資産の本拠地のため当面ここに置く。

**例外・前提**: ユーザ確認が要る編集（グローバル設定・既存 CLAUDE.md への追記）や、文脈読解が要る判断はスクリプト化せずモデルに残す。見合うのは「スクリプト化のコスト ＜ 反復回数 × 手作業コスト」のとき。1回限り・自明な処理まで無理にスクリプト化しない（オーバーエンジニアリング回避）。
