### Agent skill の冪等 setup は「初回のみ」と書かず、毎回通る存在確認ゲートにする

**学び**: 冪等な setup ステップの見出し・本文に「初回のみ」「once」等と書くと、エージェントが「もう済んだはず」と判断して存在チェックすらせず素通りし、未セットアップの環境でも初期化が走らないことがある。代わりに「毎回このステップを通る」と明記し、冪等性は **sentinel の存在チェック（ディレクトリ / 目印ファイルの有無）** で担保する。チェックは「あるか無いか」だけにして中身は検査しない（毎回走るので軽量に保つ）。

**根拠**: 2026-06 learning skill の setup 設計時。過去に別プロジェクトで「初回のみ」と書いた setup を素通りされ、repo セットアップが実行されなかった事例をユーザが報告。見出しから「（初回のみ）」を外し、「毎回このステップを通り、チェックは『あるか無いか』だけで判断する」に修正した。

**適用範囲**: 毎回呼ばれるが実体は初回だけ動けばよい、冪等な setup / bootstrap を持つ全ての agent skill（Claude Code / Codex / Pi 横断）。

**例外・前提**: 存在チェックは粗いゲート（sentinel の有無）で、中身の不整合までは検出しない。完全性より「軽量さ・確実な実行」を優先する設計を選ぶときの指針。
