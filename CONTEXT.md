# CONTEXT.md

macOS 開発環境と、複数のエージェント（Claude Code / Codex / Pi 等）の設定資産を一元管理する dotfiles リポジトリの用語集。
同じ概念が複数の語で呼ばれるのを防ぐため、正規の語を1つに固定する。用語以外（手順・設計判断）は扱わない。

## 📖 用語

### 🗂️ リポジトリの構成

**シンボリックリンク**:
リポジトリ内の実体を、ホームディレクトリなど利用側のパスから参照させるためのリンク。
_Avoid_: symlink（散文での使用。`symlink.sh` / `symlink.yml` / `task install:symlink` といった識別子は原文のまま）

**実体**:
シンボリックリンクではなく中身そのものを持つファイルやディレクトリ。リポジトリ側に1つだけ置く。

**SSoT**:
設定の唯一の正をリポジトリに集約するという設計方針。方針の名前であり、個々のファイルを指す「実体」とは別。

### 🤖 エージェント横断構成

**エージェント**:
Claude Code / Codex / Pi など、このリポジトリの設定資産を読み込んで動作する AI コーディング支援ソフトウェア本体。
_Avoid_: AIエージェント, AI エージェント, AI コーディングエージェント, ツール（エージェントを指す用法。mise や go-task のような道具一般を指す場合は対象外）

**サブエージェント**:
エージェントが処理を委譲するために内部で起動する子エージェント。単に「エージェント」とは呼ばない。

**コンテキストファイル**:
エージェントが起動時に読み込む、常時有効な指示を書いた文書。
_Avoid_: context file

**スキル**:
エージェントが必要に応じて読み込む、手順や知識をまとめた資産。
_Avoid_: Agent Skill, Skill（仕様そのものを指す「Agent Skills 標準」は除く）

**フック**:
エージェントが提供する、特定のイベントで外部処理を起動する機能。起動機構も設定形式もエージェントごとに異なる。

**共有スクリプト**:
フックから実行される、エージェントに依存しないロジックの実体。

**アダプタ**:
各エージェントのフックに登録する、共有スクリプトを呼ぶだけの薄い定義。

**Extension**:
Pi においてアダプタを実装する形態。アダプタの一種であり、フックと並列の概念ではない。
